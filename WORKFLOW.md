# Suggested Workflow

## Development cycle

1. Draft a short lesson goal.
2. Write a brief storyboard in markdown.
3. Implement one Manim scene.
4. Render a low-quality preview.
5. Review timing, readability, and math accuracy.
6. Revise the scene.
7. Render a final version only after preview approval.

## Scope control

- Build one lesson at a time.
- Do not create a large batch of scenes before the rendering pipeline is stable.
- Prefer small commits with clear messages.

## Quality control checklist

Before treating a scene as complete, verify:
- the math is correct;
- the text is readable;
- the pacing is appropriate;
- the scene name is descriptive;
- the render command works as documented.
